Overview

 

Cut the administrative burden, take advantage of our online application tool.

Allow your bursary and grants applicants to apply online, and integrate these forms into your organization workflow.

 

Topics

 

Features

•Pre-screen and auto reject non-compliant applications

•All applications and documentation are submitted directly into DevMan

•Initiate timely submissions with automated workflow reminders

•Bulk close out incomplete applications when the period is closed

•Easily report on the results of the application cycle

 

Important reminders

•Please notify the DevMan team of when the application should close.

•Closing of the application can only take place on during business hours - this is a manual step.

•It is important to bulk update all incomplete applications to closed (Declined - Incomplete).

•It is optional to send a decline notification if applications were not completed.

 

Reporting issues

Get as much information from the applicant as you can:

•Name / organisation, email address, ID number / reference number

•Description of problem

•Screenshot if possible

•Which device and browser the applicant is using

FAQ

 

My application won't submit

Check the following:

•Are all required fields completed / selected?

•Are phone numbers and email addresses in the correct format?

•Are all required attachments uploaded?

•If Yes to either question, please check which browser the applicant is using.

•If the issue persists using Chrome or Edge, request information as noted in the Reporting issues section, and send to the support desk.

 

My application was rejected / kicked me out

Find the application is DevMan and check the status.

•If declined, the applicant has not met the criteria. If an answer was submitted incorrectly the applicant should start again. (See the Re-open an application tutorial.)

•If phase 1 was successful, check if the email with the link to phase 2 was sent successfully.

•If not, follow the steps on how to correct the email address and send the email with the link. (See the Re-send a failed email tutorial.)

 

The online page is not loading / fields do not show options

•Is the applicant using Chrome or Edge?

•If no, direct the applicant to try Chrome or Edge.

•If yes, request information as noted in Reporting issues and send to the support desk.

 

I attached the wrong documents or did not fully answer a question in phase 2

•Delete the fully completed milestone on the project / bursary.

•The applicant can use the link in the phase 2 email to go back and update their application. (View the second part of the tutorial, Re-open an application.)
