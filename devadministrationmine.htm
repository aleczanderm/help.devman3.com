Overview

 

We cannot afford to be complacent or careless about computer security and the safety of the data. Computers and servers are being hacked all the time, for the data on them and to abuse in some other way.

DevMan provides the means to provide controlled access to your data to those authorised to use it. It does however require your active support and vigilance.

 

Topics

 

Features

•User - Access to the functions in Devman is limited to persons who are defined as user. Users must enter a userid (username) and password.

•Roles - What information a user is able to see and work with is managed in roles and privileges.

•A record with the name and date and time is made each time a person logs onto DevMan.

•Audit logging - Creates a day to day "audit-trail" of activity (when, what and by whom) on the records in your database.

 

User roles & permissions

 

Setting up roles

To control access to data, you can modify existing roles, create new roles, or change which security roles are assigned to each user.These privileges (task-based and record-level privileges) are cumulative i.e. users have each privilege you make available to them.

 

Permissions

1. Select Security > Users.

2.Select Add for a new user or select from the user list if changing an existing person.

 

3. Enter the user's credentials, and select the role based rights:

•None    - cannot see data in a role

•View     - view only

•Edit        - view and edit

•Add       - view, edit and add a new record

•Admin  - complete control including delete

 

 

3. Select Save > Save and close and return to the System user rights list grid where the new permission will show.

 

NOTE:  You can only alter a user’s rights if you have permission to do so.

User profile

To access the security module, select Security > Users.  

 

Add and link a new DevMan user

1.Add a person profile for the new user.

2.Select Save and stay on page. Update any additional details.

3.Click on System status then select Internal. Select Save and then select Save and close.

 

4.Add a user profile. Select Administration > Security > Users. Adding a user profile and linking it to the person profile

•links the person's email address to emails that will be sent.

•links the contact details for the merge tabs for the Activities by list.

 

5.Select Add and complete the user profile.

 

6.Select Save and stay on page to update additional details and select the applicable security mode. This allows users to either have no rights, view rights, edit rights, add rights or admin rights (all).

 

7.Select the applicable Permissions for your user. Then select Save and close.
Change preferences

You can change preferences:

•From the user's profile select Preferences from the sub menu.

•From the person's profile select Maintenance > Preferences.

 

Verify sender email address

To verify the email address, Click on the user again, then click on the notepad next to the email address. Select Request. Once you receive an email, select Verify.
