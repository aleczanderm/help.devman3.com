User

The term is usually well-known. In the context of The Development Manager program it denotes a person who can "use" or is able get access to the program via a login process and perform the functions provided.

 

The user must:

 a) be added to a database and given a "user-name" and password.

 b) assigned a level of access to the activities they can perform.  
 

Kidz will get this started by adding your initial set of users with permissions. A "power-user" will be assigned who can maintain the list of users and permissions. In addition the power-user can assist you when you have problems logging in or when you may have forgotten your password.

 

Roles, Functions, Modules and action levels.

Persons are assigned to a role. Each role is assigned to a number of modules and within each module there are a number of actions it can perform.
