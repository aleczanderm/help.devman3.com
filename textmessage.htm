Overview

 

Mobile communication is a key platform for stakeholder communication, often used to augment existing email based communication, and sometimes even replacing it, especially for younger audiences.

 

DevMan and its partners work closely together to ensure that the system keeps up with the fast pace of new product offerings. This includes  adhering to the WASPA code of conduct.

 

Topics

 

       Features

•Make use of templates to control communication styles

•Add merge tags for personalisation

•SMS delivery status – See whether an SMS was delivered

•Links inside an SMS can be shortened using a shortening service (characters are limited to 160)

 

       SMS delivery status

Track the success or failure of your SMS:

 

Sent

Successfully submitted to the network.

 

Delivered

Delivery confirmed by handset.

 

Responded

Replied to the message. The response can be viewed in the delivery details of that particular SMS.

 

Failed

Number does not exist or is invalid.

 

Undeliverable

Network Indicated the message cannot be delivered.

 

       Guidelines

•Text messages to International numbers and local text messages longer than 160 characters are treated as multiple credits.

•Sending bulk messages via a prepared template is considered best practice (Contact the Support Team for assistance)

•Sending bulk messages with an embedded URL must be sent via a template.

•When sending a mailshot, a pop up notification might advise of the number of credits available. If insufficient credit is available, you can still send the message as it will be placed in the queue - our Support Team regularly loads credits.

 

       Send SMS

Send from various parts of the system transactionally or in bulk

•Contact profiles or lists

•Project and bursary profiles or lists

•From an interaction

•Via workflow - automatic or manual

•Mailshots and events

 

       Steps

•From the profile page select Action > Send SMS

•From a grid select Send SMS / Send communication > Send SMS

•From an interaction select Send SMS

•From the Communication page

 

       Tip

DevMan will check for non-supported characters before sending the SMS and make adjustments.

 

       Manage SMS

 

       Check delivery status

From the Communication dashboard, select SMS > View, then select Itemised list. If you wish, you can now filter on Sender / Status and you will be able to see the Delivery status for each SMS sent.

 

 

       View sent SMS

There are a number of ways to view a sent SMS

•From the person / organisation page, select View > Communication. Filter on SMS. From the list displayed, click on the one that you want to view. If the SMS you want to view is showing on the profile summary page, you can click on it to open it.

•From the Communication tab, select SMS > View then select Itemised list. Here you can filter by Date sent / Recipient etc. and then click on the SMS you want to view.

 

       Manage invalid numbers

Select Communication > SMS > Problem Numbers

 

Update the numbers on the related person profile.

 

 

       Bulk import

If you have a spreadsheet of numbers that are not on DevMan, it is possible to import the list and bulk send an SMS. Please contact the Support Team for assistance.
